# stashcli — فضای ذخیره‌سازی ابری با تلگرام

فایل‌های خود را با استفاده از اکانت تلگرام‌تان به‌عنوان بک‌اند ذخیره‌سازی،
نگه‌داری، همگام‌سازی و استریم کنید. می‌توانید آن را به‌صورت درایو FTP یا
WebDAV مانت کنید، یا مستقیماً از خط فرمان استفاده کنید — فایل‌ها به
تکه‌های رمزنگاری‌شده تقسیم و به‌صورت پیام‌های تلگرام آپلود می‌شوند و هنگام
دانلود دوباره کنار هم چیده می‌شوند.

- **فایل‌های بزرگ در کمترین تعداد تکه ممکن ذخیره می‌شوند.** به‌طور پیش‌فرض
  هر فایل در تکه‌های ثابت **۴۵۰ مگابایتی** آپلود می‌شود — یک فیلم ۸۰
  مگابایتی به‌جای ده‌ها پیام کوچک، فقط در **یک** پیام آپلود می‌شود.
- **به‌طور پیش‌فرض سریع.** آپلود و دانلود بدون هیچ تنظیمی به‌صورت
  پیش‌فرض با **۴ تکه هم‌زمان** انجام می‌شود.
- **قابل ازسرگیری (Resumable).** قطعی اتصال یا بسته‌شدن ناگهانی برنامه
  باعث از دست رفتن کل انتقال نمی‌شود — همان دستور را دوباره اجرا کنید تا
  از همان‌جا ادامه پیدا کند.
- **فایل فقط بعد از تکمیل کامل آپلود نمایش داده می‌شود.** هیچ ورودی
  ناقص/نیمه‌کاره‌ای در `ls`، FTP یا WebDAV در حین انتقال دیده نمی‌شود.
- **چندسکویی (Cross-platform).** روی ویندوز، لینوکس و مک از یک فایل
  اجرایی ساده کار می‌کند.

## ۱. نصب

به [Go نسخه ۱.۲۱ یا بالاتر](https://go.dev/dl/) نیاز دارید. تنها پیش‌نیاز
ساخت پروژه همین است.

```bash
git clone https://github.com/yourname/stashcli.git
cd stashcli
go build -o stashcli .
```

همین! حالا یک فایل اجرایی `stashcli` (یا `stashcli.exe` در ویندوز) در
همین پوشه دارید. نیازی به Docker یا هیچ سرویس خارجی دیگری نیست.

### ساخت برای سیستم‌عامل دیگر (کراس‌کامپایل)

Go می‌تواند بدون نیاز به نصب آن سیستم‌عامل، برای آن باینری بسازد:

```bash
# ویندوز (از لینوکس/مک)
GOOS=windows GOARCH=amd64 go build -o stashcli.exe .

# لینوکس (از ویندوز/مک)
GOOS=linux GOARCH=amd64 go build -o stashcli .

# مک (Apple Silicon)
GOOS=darwin GOARCH=arm64 go build -o stashcli .
```

## ۲. گرفتن API ID و API Hash تلگرام

۱. به آدرس <https://my.telegram.org> بروید و با شماره تلفن خود وارد شوید.
۲. بخش **API development tools** را باز کرده و یک اپلیکیشن بسازید (هر
   نامی مناسب است).
۳. مقادیر **api_id** و **api_hash** نمایش داده‌شده را کپی کنید — در مرحله
   بعد به آن‌ها نیاز دارید.

## ۳. پیکربندی

فایل نمونه تنظیمات را کپی کرده و اطلاعات API خود را وارد کنید:

```bash
cp settings.example.json settings.json
```

```jsonc
{
  "uploadchunksize": 471859200,   // ۴۵۰ مگابایت — همین‌طور بگذارید، یا در اینترنت ناپایدار کمترش کنید
  "api_id": 123456,               // از my.telegram.org
  "api_hash": "your_api_hash",    // از my.telegram.org
  "parral_download": 4,           // تعداد تکه‌های دانلود هم‌زمان
  "parral_upload": 4,             // تعداد تکه‌های آپلود هم‌زمان
  "cache_max_size_mb": 200,
  "cache_expire_days": 7
}
```

نیازی نیست حتماً `uploadchunksize`/`parral_upload`/`parral_download` را
تنظیم کنید — اگر خالی بگذارید (یا `0`)، stashcli خودش به‌صورت پیش‌فرض از
تکه‌های ۴۵۰ مگابایتی و موازی‌سازی ۴ تایی استفاده می‌کند.

> **اینترنت شما کند یا ناپایدار است؟** مقدار `uploadchunksize` را کوچک‌تر
> بگذارید، مثلاً `16777216` (۱۶ مگابایت) یا `67108864` (۶۴ مگابایت).
> تکه‌های بزرگ‌تر روی اینترنت خوب سریع‌تر هستند، اما اگر یک تکه قطع شود
> باید کل همان تکه دوباره فرستاده شود، پس تکه‌های کوچک‌تر روی اینترنت
> ناپایدار امن‌ترند.

سپس پایگاه‌داده (خالی) ذخیره‌سازی را بسازید و یک‌بار وارد حساب شوید:

```bash
./stashcli storage gen storage.json
./stashcli storage add
```

دستور `storage add` شماره تلفن شما (برای ورود به تلگرام) و یک یا چند
**شناسه چت (chat ID)** را می‌پرسد — این‌ها همان جاهایی هستند که تکه‌های
فایل شما به آن‌جا فرستاده می‌شود (یک گروه خصوصی یا «پیام‌های ذخیره‌شده» هر
دو مناسب‌اند). می‌توانید شناسه چت را از هر ربات تلگرامی که آن را گزارش
می‌دهد، یا با فوروارد کردن یک پیام از آن چت به `@userinfobot`/`@getidsbot`
بگیرید.

> فایل `storage.json` نشست فعال تلگرام شما را در خود دارد — با آن مثل یک
> رمز عبور رفتار کنید. هرگز آن را commit یا به اشتراک نگذارید.

## ۴. استفاده

### مستقیم از خط فرمان

```bash
# آپلود
./stashcli upload ./movie.mkv /movies/movie.mkv

# دانلود
./stashcli download /movies/movie.mkv ./movie.mkv

# نمایش فهرست یک پوشه
./stashcli ls /movies

# دیدن اطلاعات: حجم/تاریخ آپلود برای یک فایل، یا تعداد آیتم‌ها برای پوشه
./stashcli info /movies/movie.mkv
./stashcli info /movies

# ساخت پوشه، حذف فایل/پوشه
./stashcli mkdir /movies
./stashcli rm /movies/movie.mkv
```

### مانت به‌عنوان درایو (WebDAV)

```bash
./stashcli webdav --port 8080 --user alice --pass secret
```

سپس از این‌جا وصل شوید:
- **ویندوز:** File Explorer ← *This PC* ← *Map network drive* ← `http://127.0.0.1:8080`
- **مک:** Finder ← *Go* ← *Connect to Server* ← `http://127.0.0.1:8080`
- **لینوکس:** هر کلاینت WebDAV، یا با `davfs2`/`rclone mount` مانت کنید

### مانت به‌عنوان FTP

```bash
./stashcli ftp --port 21 --user alice --pass secret
```

با هر کلاینت FTP (FileZilla، WinSCP، دستور `ftp`، کلاینت پیش‌فرض سیستم‌عامل
و...) وصل شوید.

### استریم مستقیم یک فایل (بدون نیاز به دانلود)

```bash
./stashcli stream --port 8081
```

سپس آدرس `http://127.0.0.1:8081/movies/movie.mkv` را در مرورگر یا یک
پخش‌کننده مثل VLC/mpv باز کنید — قابلیت پرش/جلو-عقب بردن بدون دانلود کامل
فایل را دارد.

## ۵. اجرا به‌عنوان یک سرویس پس‌زمینه

### لینوکس (systemd)

فایل `/etc/systemd/system/stashcli-webdav.service` را بسازید:

```ini
[Unit]
Description=stashcli WebDAV server
After=network.target

[Service]
WorkingDirectory=/opt/stashcli
ExecStart=/opt/stashcli/stashcli webdav --port 8080 --user alice --pass secret
Restart=on-failure
User=youruser

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now stashcli-webdav
```

### ویندوز

آن را از یک فایل `.bat` اجرا کنید، یا با [NSSM](https://nssm.cc/) به‌عنوان
یک سرویس واقعی ویندوز نصبش کنید تا بعد از ریستارت هم فعال بماند.

## نکات امنیتی

- هر تکه پیش از آپلود با یک رمز تصادفی مخصوص به خودش رمزنگاری می‌شود
  (AES-256-GCM) — تلگرام هرگز محتوای واقعی فایل شما را نمی‌بیند.
- سرورهای FTP/WebDAV/استریم **رمزنگاری انتقال داخلی ندارند**. برای دسترسی
  از راه دور، آن‌ها را روی `127.0.0.1` نگه دارید و از تونل SSH یا یک
  reverse proxy با TLS استفاده کنید. اگر روی چیزی غیر از `127.0.0.1` بایند
  می‌کنید، حتماً `--user`/`--pass` را تنظیم کنید.
- فایل `storage.json` رشته نشست فعال تلگرام شما را نگه می‌دارد — هرکسی
  که آن فایل را داشته باشد دسترسی کامل به حساب شما دارد. آن را جایی
  خصوصی نگه‌داری کنید و هرگز در گیت commit نکنید.

## پشت یک شبکه فیلترشده/سانسورشده هستید؟

stashcli می‌تواند اتصال تلگرام خود را از طریق یک پراکسی SOCKS5 یا MTProto
عبور دهد — به فیلد `proxy` که در `configs.go` مستند شده مراجعه کنید، یا
`"proxy": {"type": "system"}` را در `settings.json` قرار دهید تا پراکسی
SOCKS تنظیم‌شده در سیستم‌عامل شما (ویندوز و لینوکس/GNOME/KDE پشتیبانی
می‌شوند) به‌طور خودکار شناسایی شود.
